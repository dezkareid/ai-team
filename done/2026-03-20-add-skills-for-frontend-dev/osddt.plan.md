# Implementation Plan: Frontend Development Skills

## Architecture Overview
The project follows a source-and-export model. Skills are maintained as `SKILL.md` files (and supporting references) in the `skills/` directory. The `.agent-structurerc` file maps these skills to specific Claude Code plugins. The build process (`pnpm run export-claude`) then generates the specialized Markdown files for Claude in the `plugins/` directory.

We will introduce a new Claude plugin named `frontend-tools` and add the new skills (`react-best-practices`, `next-best-practices`) to it.

## Implementation Phases

### Phase 1: Skill Acquisition
- **Goal**: Import existing high-quality skills from external repositories.
- **Steps**:
  - Run `npx skills add react-best-practices --from vercel-labs/agent-skills --agent claude --symlink`.
  - Run `npx skills add next-best-practices --from vercel-labs/next-skills --agent claude --symlink`.
  - Verify the skills are added to the local `skills/` directory.

### Phase 2: Configuration Integration
- **Goal**: Register the new plugin and skills in the project's metadata.
- **Steps**:
  - Update `.agent-structurerc` to add the `frontend-tools` plugin to `claude-plugins`.
  - Update `.agent-structurerc` to map `react-best-practices` and `next-best-practices` to the `frontend-tools` plugin.
  - Update any version numbers or descriptions as needed.

### Phase 3: Cleanup and Export
- **Goal**: Remove redundant files and synchronize the project.
- **Steps**:
  - Delete any Claude-specific configuration files generated by the `npx skills` command (e.g., `.claude/` if it was created in the source folders).
  - Run `pnpm run build` to ensure all scripts are up to date.
  - Run `pnpm run export-claude` to generate the new plugin structure in the `plugins/` directory.
  - Run `pnpm run distribute-mcp` to ensure the Gemini extension configuration is updated.

## Technical Dependencies
- `npx skills`: Tool for fetching and managing agent skills.
- `pnpm`: Project package manager for build and export tasks.

## Risks & Mitigations
- **Command Failure**: If `npx skills` doesn't work as expected, I will manually inspect the command's help or repository to find the correct flags.
- **Plugin Structure Mismatch**: I will carefully review the generated `plugins/frontend-tools` directory against existing plugins to ensure it follows the project's conventions.

## Out of Scope
- Modifying the content of the imported skills.
- Creating new commands for the `frontend-tools` plugin.
